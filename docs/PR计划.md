# PR 计划

本项目按照小粒度 PR 持续提交。每个 PR 只完成一个独立目标，合并后主分支应保持可运行或可预览。

## PR 1：初始化项目结构与中文 README 文档

### 功能描述

建立项目基础说明、方案设计、PR 计划和目录结构，明确 Windows 桌面端 AI 同声传译助手的目标、技术路线和后续开发节奏。

### 实现思路

先完成中文 README 和 docs 文档，说明系统音频捕获、悬浮字幕、上下文翻译、字幕修正和术语一致性的设计。

### 测试方式

检查 README 和 docs 文档能否在 GitHub 正常预览，确认目录结构清晰，文档内容与比赛题目要求一致。

## PR 2：添加桌面应用基础入口

### 功能描述

添加 Python 应用入口和基础配置加载逻辑，为后续 UI 和 AI 引擎接入提供运行骨架。

### 实现思路

建立 `app` 包结构，提供主程序入口、配置对象和基础日志输出。

### 测试方式

在 Windows 环境执行启动命令，确认程序可以正常启动并输出基础状态信息。

## PR 3：添加主控制窗口

### 功能描述

实现桌面主控制窗口，提供开始、暂停、停止、状态展示和基础配置入口。

### 实现思路

使用 PySide6 / Qt Quick 构建现代工具型界面，UI 线程与后台任务分离。

### 测试方式

启动应用，检查窗口显示、按钮状态切换和基础交互是否正常。

## PR 4：添加悬浮字幕窗口

### 功能描述

实现可置顶、可拖动、可调整透明度和字号的悬浮字幕窗口。

### 实现思路

使用独立窗口展示实时字幕，支持中文单语和双语模式，并预留字幕状态样式。

### 测试方式

启动应用后显示悬浮窗，检查置顶、拖动、透明度和字号调节是否正常。

## PR 5：定义字幕事件协议和状态管理

### 功能描述

定义 `partial`、`final`、`updated` 三类字幕状态，并支持根据 `segment_id` 更新历史字幕。

### 实现思路

建立字幕事件模型和字幕状态容器，确保修正事件不会追加重复字幕，而是回写指定字幕段。

### 测试方式

使用单元测试或模拟事件验证新增、确认和修正字幕的状态变更。

## PR 6：添加模拟字幕流

### 功能描述

使用模拟字幕事件驱动主窗口和悬浮窗，提前验证实时字幕展示和历史回修体验。

### 实现思路

内置一组英文技术分享示例，按时间间隔发送临时字幕、正式字幕和修正字幕事件。

### 测试方式

启动模拟模式，观察字幕是否持续更新，历史字幕修正时是否正确高亮。

## PR 7：实现 Windows 系统音频捕获

### 功能描述

通过 Windows WASAPI Loopback 捕获当前系统播放声音，并支持保存短音频用于测试。

### 实现思路

封装音频采集器，持续获取默认播放设备输出音频，输出统一格式的音频帧。

### 测试方式

播放本地视频或浏览器音频，启动采集后保存测试 wav，确认录到的是系统声音。

## PR 8：添加音频处理与 VAD 分段

### 功能描述

添加音频切片、重采样、滑动窗口缓存和语音活动检测能力。

### 实现思路

将连续音频流整理为适合 ASR 的片段，并根据静音判断字幕段边界。

### 测试方式

使用录制音频验证切片、缓存和分段结果是否符合预期。

## PR 9：接入语音识别模块

### 功能描述

接入真实 AI ASR 能力，将系统音频流转换为原文文本，并把临时识别结果和最终识别结果接入桌面字幕 UI。

### 实现思路

通过适配器封装 ASR 调用，保留 mock 模式，便于无模型或无 API Key 时演示 UI。真实 ASR 采用百度智能云实时语音识别 WebSocket API，在 VAD 触发 `speech_start` 后建立会话并持续发送 16k 单声道 PCM 音频帧，解析 `MID_TEXT` 临时识别结果和 `FIN_TEXT` 最终识别结果。桌面 UI 使用后台线程消费有界音频队列，避免录音、识别和界面刷新互相阻塞。

### 测试方式

使用英文测试音频验证 mock 与真实 ASR 调用入口；播放系统音频后启动桌面端，检查 `MID_TEXT` 是否作为临时字幕刷新、`FIN_TEXT` 是否确认当前字幕段；检查未配置 API Key、服务返回异常或音频格式不支持时是否能给出清晰提示。

## PR 10：接入中文翻译模块

### 功能描述

将识别出的外语文本翻译为中文，并支持结合上下文进行翻译。

### 实现思路

通过翻译适配器接入大模型接口，输入当前文本、最近上下文和术语表。

### 测试方式

使用固定英文片段测试翻译结果，验证术语翻译和上下文表达是否稳定。

## PR 11：实现历史字幕自动修正

### 功能描述

根据后续上下文自动判断并修正最近字幕段中的识别或翻译错误。

### 实现思路

保留最近若干条字幕记录，触发修正检查后生成 `segment.update` 事件并回写 UI。

### 测试方式

使用包含术语歧义的示例验证字幕从错误表达修正为更准确表达。
